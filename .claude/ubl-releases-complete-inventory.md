# UBL Complete Release Inventory

This document contains the complete chronological inventory of all UBL releases from the OASIS UBL Technical Committee, starting from UBL 1.0 (2004) through UBL 2.5 (draft).

## Release Types Legend

- **cd** = Committee Draft
- **prd** = Public Review Draft (predecessor to csprd)
- **csprd** = Committee Specification Public Review Draft
- **csd** = Committee Specification Draft
- **cs** = Committee Specification
- **cos** = Committee OASIS Specification
- **os** = OASIS Standard (official final release)

## Release Naming Convention Notes

1. **Starting from UBL 2.2**, OASIS changed from numbered suffixes (prd1, prd2) to zero-padded suffixes (csprd01, cs01)
2. **errata-UBL-2.0** (entry #10) - Draft corrections package; SKIP in import (superseded by #11)
3. **os-UBL-2.0-update** (entry #11) - Final approved delta/patch package; apply as PATCH on top of #9
4. Some intermediate stages don't have packages in subdirectories but may have root-level ZIPs

The following packages require special handling as they are NOT full releases:

## Recommended Commit Strategy

For building git history:
1. Commit infrastructure code first (Python scripts, this manifest)
2. Commit each UBL release in chronological order (1-37), with special handling:
   - **Regular releases (#1-9, #12-37):** Extract full package, commit all files
   - **Patch packages (#11):** Extract and overlay on previous release, commit only changed files
   - **Skipped packages (#10):** Do not import
3. Tag each "os-" (OASIS Standard) release with version tags (e.g., `v2.0`, `v2.1`)
4. Tag intermediate releases with full names (e.g., `prd3-UBL-2.0`, `cs01-UBL-2.2`)
5. Tag patch/errata releases with descriptive names (e.g., `os-UBL-2.0-errata01`, `v2.0-errata01`)

### Import Tool Requirements

When creating the import tool, implement package type detection:

```python
def get_package_type(package_name, package_url):
    """Determine how to handle this package."""
    if package_name == "errata-UBL-2.0":
        return "SKIP"  # Draft version, superseded by update package
    elif "update-delta" in package_url or package_name.startswith("errata-"):
        return "PATCH"  # Apply as overlay on previous release
    else:
        return "FULL"   # Regular full release import
```

For PATCH-type packages:
1. Download and extract to temporary location
2. Identify base release (previous OASIS Standard)
3. Copy extracted files over base release files
4. Use `git add` only on modified files (git will auto-detect changes)
5. Commit with special message format indicating correction/errata nature
6. Tag appropriately

## Directory Structure for Each Release

Most releases contain:
- **xsd/** - XML Schema Definitions (normative)
- **xsdrt/** - Runtime schemas (annotations stripped)
- **cl/** - Code lists

- **art/** - Diagrams and artwork
- **db/** - Database representations
- **cva/** - Context/Value Association rules (2.1+)
- **rnc/** - RELAX NG schemas (2.1+)
